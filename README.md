## 1. Введение

### 1.1. Назначение документа

Описывает функциональные и нефункциональные требования к MVP системы ипотечного кредитования. Ипотека рассматривается как единый продукт без разделения на целевые, социальные или ведомственные программы. Описывается универсальный жизненный цикл: подача заявки → скоринг → одобрение → выдача → сопровождение.

### 1.2. Целевая аудитория

- Физические лица (заёмщики)
- Ипотечные менеджеры банка
- Администраторы системы

### 1.3. Область действия (MVP)

**Входит в MVP:**
- Онлайн-анкетирование с автозаполнением из ЕСИА
- Загрузка документов (2-НДФЛ — обязательно)
- Автоматический бинарный скоринг
- Интеграция с БКИ
- Управление статусами заявки
- Аннуитетный график платежей
- Личные кабинеты заёмщика и менеджера
- Остаток долга в ЛК

## 3. Функциональные требования

### 3.1. Заявка и анкетирование

| ID | Требование | Критерий приёмки |
|----|------------|------------------|
| ФТ-1 | Система предоставляет многошаговую форму сбора данных: личные данные, доход и занятость, параметры кредита (сумма, срок, первоначальный взнос), объект недвижимости | Заёмщик заполняет анкету за 5–7 минут, переход между шагами без потери данных |
| ФТ-1.1 | После успешной аутентификации через ЕСИА система автоматически получает и подставляет в анкету подтверждённые данные: ФИО, дата рождения, паспорт (серия, номер, кем выдан, дата выдачи), ИНН, СНИЛС, адрес регистрации. Заёмщик не может редактировать эти поля | При входе через ЕСИА поля заполняются автоматически, кнопка «Отправить» активна только при успешном получении данных |
| ФТ-1.2 | В разделе «Объект недвижимости» заёмщик обязан указать: адрес (с валидацией через ФИАС), тип недвижимости (квартира/дом/таунхаус/апартаменты), стоимость объекта (от 500 тыс. до 100 млн ₽) | Без заполнения этих полей отправка заявки невозможна |
| ФТ-2 | Система выполняет валидацию полей в реальном времени: ИНН (12 цифр), телефон (10 цифр после +7), сумма кредита (300 тыс. – 30 млн ₽), возраст заёмщика (21–65 лет на момент окончания кредита) | При невалидных данных кнопка «Отправить» неактивна, поле подсвечивается красным с текстом ошибки |
| ФТ-3 | Система автоматически сохраняет черновик заявки в localStorage браузера. Данные восстанавливаются при перезагрузке страницы. При закрытии вкладки или очистке кэша черновик теряется | При перезагрузке страницы ранее введённые данные отображаются в форме |

### 3.2. Скоринг и верификация

| ID | Требование | Критерий приёмки |
|----|------------|------------------|
| ФТ-4 | После отправки заявки система направляет запрос в БКИ для получения кредитной истории. Таймаут ожидания ответа — 5 секунд | Запрос логируется с timestamp. При таймауте заявка переходит в статус «Ошибка БКИ» |
| ФТ-4.1 | При ошибке БКИ (таймаут, отказ, 5xx) менеджер может вручную повторить запрос. При 4xx ошибках повтор невозможен, заявка переводится в статус «Отказана» с комментарием | В карточке заявки есть кнопка «Повторить запрос БКИ». Каждый повтор логируется |
| ФТ-5 | Система рассчитывает скоринг-балл и принимает бинарное решение. Правила расчёта (веса и пороговые значения настраиваются администратором): <br> - ПДН (платёж/доход) — вес 50% <br> - Кредитная история (отсутствие просрочек > 90 дней за последние 12 месяцев) — вес 30% <br> - Стаж на текущей работе — вес 20% <br><br> Решение: если балл ≥ порогового значения → «Одобрена», иначе → «Отказана». Начальные настройки: порог = 450, ПДН ≤ 50%, стаж ≥ 3 месяцев | Скоринг выполняется только после загрузки справки 2-НДФЛ. Результат отображается через UI ≤ 15 секунд после отправки |
| ФТ-5.1 | Администратор изменяет через интерфейс: веса факторов, пороговое значение балла, максимальный ПДН, минимальный стаж | Изменения вступают в силу без перезапуска системы, логируются с указанием администратора |

### 3.3. Документооборот и статусы

| ID | Требование | Критерий приёмки |
|----|------------|------------------|
| ФТ-7 | Система позволяет заёмщику загружать документы: <br> - Справка 2-НДФЛ (за последние 12 месяцев) — обязательна <br> - Скан паспорта — опционально <br> - Фото объекта недвижимости — опционально <br><br> Форматы: PDF, JPG, PNG. Размер: до 10 МБ на файл. Максимум 5 файлов на заявку | Без загруженной справки 2-НДФЛ кнопка «Отправить заявку» неактивна |
| ФТ-8 | Жизненный цикл заявки: <br> **Черновик** → **На рассмотрении** → **Запрос документов** → **Одобрена** / **Отказана** → **Выдана** <br><br> Дополнительный статус: **Ошибка БКИ** | Каждое изменение статуса логируется с timestamp, пользователем и комментарием |
| ФТ-9 | При смене статуса заявки на главной странице личного кабинета заёмщика отображается баннер. Текущий статус всегда виден в шапке ЛК | Баннер появляется при следующем входе заёмщика в ЛК после смены статуса |

### 3.4. График платежей и обслуживание

| ID | Требование | Критерий приёмки |
|----|------------|------------------|
| ФТ-10 | При одобрении заявки система формирует аннуитетный график платежей. График содержит: дату платежа, сумму платежа, сумму основного долга, сумму процентов, остаток долга после платежа | График отображается в ЛК заёмщика в табличном виде |
| ФТ-11 | Заёмщик в личном кабинете видит текущий остаток долга по кредиту | Остаток отображается на главной странице ЛК, обновляется после каждого платежа |

## 4. Нефункциональные требования

### 4.1. Производительность

| ID | Требование | Критерий приёмки |
|----|-------------|------------------|
| НФТ-1 | Время загрузки анкеты и графика платежей ≤ 2 секунд для 95% запросов | Таймирование в логах каждого запроса |
| НФТ-2 | Расчёт скоринга + получение данных из БКИ ≤ 15 секунд. Заёмщик видит индикатор загрузки и сообщение «Идёт проверка, мы уведомим вас в личном кабинете» | Таймирование в логах каждого запроса |
| НФТ-3 | Система выдерживает до 500 одновременных сессий заёмщиков без деградации производительности | Нагрузочное тестирование (JMeter, 500 concurrent users) |

### 4.2. Безопасность и соответствие 152-ФЗ

| ID | Требование | Критерий приёмки |
|----|-------------|------------------|
| НФТ-4 | Все персональные данные шифруются: в транзите — TLS 1.3, в покое — AES-256 | Результаты пентеста: отсутствие plaintext ПДн в логах, дампах БД, сетевом трафике |
| НФТ-5 | Система соответствует 152-ФЗ: <br> - Логирование каждого доступа к ПДн (кто, когда, к каким данным) <br> - Возможность отзыва согласия на обработку ПДн <br> - Хранение всех данных в пределах РФ | При отзыве согласия заёмщиком в ЛК: заявка переводится в статус «Архив (обезличено)», ПДн заменяются на хэш (SHA-256), исходные данные удаляются в течение 24 часов |
| НФТ-6 | Аутентификация: <br> - Заёмщик — через ЕСИА (Госуслуги) с получением подтверждённых персональных данных <br> - Сотрудники банка — через внутренний LDAP/Active Directory <br> - Для служебных аккаунтов — 2FA (TOTP) | Успешная авторизация через тестовый контур ЕСИА. JWT-токен валидируется на бэкенде |

### 4.3. Надёжность и мониторинг

| ID | Требование | Критерий приёмки |
|----|-------------|------------------|
| НФТ-7 | Доступность сервиса (SLA) ≥ 99.5% в месяц | Отчёт по аптайму за месяц (исключая плановые работы с уведомлением за 3 дня) |
| НФТ-8 | Автоматическое резервное копирование БД каждые 6 часов. Срок хранения бэкапов — 30 дней | Успешное восстановление БД на тестовом стенде из случайно выбранного бэкапа |
| НФТ-9 | Интеграция с системой мониторинга: алерты при падении API БКИ, ошибках валидации (более 5% от всех запросов), аномальной нагрузке (> 600 RPS) | Настроены алерты, уведомления в Telegram-канал дежурной смены |
